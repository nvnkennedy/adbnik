Camera tab

The Camera tab provides a full-area live preview (typically via OpenCV when opencv-python-headless is installed), with Windows Camera–style translucent side rails for controls. Photos (JPEG) and videos (MP4) save to a folder you choose—same as other tabs, nothing runs on a server.

Layout

  • Top strip — Short status line only (aligned with the rest of the tab area so you do not get stray light bands under the tab bar).
  • Preview — Fills the tab; preview scaling stays on the GPU-friendly path for responsiveness.
  • Left rail — ⚙ menu (start/stop preview, pause, save folder, restart), pause button, and read-only hints (WB / Focus / ISO · auto) reflecting typical consumer camera UI.
  • Right rail — Switch camera (cycles detected devices), Photo / Video mode, large primary button (shutter or record/stop), and a thumbnail of the last saved photo.

Actions

  • Primary button — If preview is off, starts preview when a device is available. In photo mode, captures a JPEG. In video mode, starts or stops MP4 recording.
  • Switch camera — Moves to the next detected camera (multiple USB webcams or built-in + external).
  • ⚙ menu — Explicit start/stop, pause, choose save folder (native folder picker on Windows), restart pipeline.
  • Thumbnail — Opens the last saved photo in your default viewer.

Performance

Preview uses a background capture thread (small buffer, drained frames on open, capped preview resolution) so the rest of Adbnik stays responsive. When you leave the Camera tab, preview pauses automatically.

Requirements

Best results: install opencv-python-headless (already a dependency of the adbnik package). If OpenCV is unavailable, Qt Multimedia may be used where your PyQt5 build supports it.
