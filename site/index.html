One workspace for ADB, SSH, serial, files, and screen

Adbnik is a single desktop app that keeps Android debugging, SSH, serial/COM, and local shells in one tabbed terminal, next to on-device file browsing and USB screen forwarding — so you spend less time switching windows and retyping connection details.

Install with pip View on PyPI

Why Adbnik

Many setups give you a terminal here, a file window there, and another place to start mirroring. Adbnik is aimed at people who routinely mix phone or tablet work with SSH to servers or serial to hardware — and want that work in one consistent layout with a shared device bar, bookmarks, and session list.

  • Unified workspace: ADB shell tabs, SSH tabs, serial tabs, and local CMD/PowerShell sessions live side by side; file access and screen forwarding stay in the same app instead of scattered tools.
  • Flows that match real work: Open SSH using the same host details you already entered for remote file access; keep bookmarks for devices and sessions you use every day.
  • Focused scope: Adbnik does not try to be a full remote-desktop suite — it focuses on terminal, device files, and display forwarding for developers and integrators who already use standard command-line components on their machine.
  • PyPI install: pip install adbnik — run adbnik.

Features

ADB & Android shell

Pick a device, open an interactive ADB shell tab, and use bookmarks and shortcuts for common tasks (root, remount, reboot, and more).

SSH terminal

Open SSH sessions from scratch or reuse remote file-access fields from the explorer. Uses the SSH client on your PATH (ssh).

Serial / COM console

Attach to COM ports at your chosen baud rate for boards and firmware logs — serial tabs alongside ADB and SSH in the same workspace.

File explorer

Browse Android device storage and transfer files; remote host fields align with SSH when you work across device and server.

Screen mirroring

Start a live USB display session from the app so you can see and control the device on your PC. You install the display-forwarding utility yourself and configure its path in the app.

Android device features need Google’s platform-tools (ADB) and a separate USB display-forwarding install, both on your PATH. SSH needs an ssh client on your PATH. Serial uses COM hardware; set port and baud in the app or preferences.
